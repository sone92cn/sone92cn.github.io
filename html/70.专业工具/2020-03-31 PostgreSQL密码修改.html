PostgreSQL密码修改

方式1：

使用psql，连接到Postgres Server：

username=> \password
Enter new password:
Enter it again:
username=>

这种修改方式相当于向postgres server 发送了如下命令：

ALTER USER postgres PASSWORD 'md53175af154as54df5as4d5f45sd6af'

方式2：

可以直接执行sql修改，可用任意客户端，如pgadmin等：

ALTER USER test1 PASSWORD 'secret'

弊端：通过sql修改，有可能会将修改语句记录在相关工具的log里; 通过psql 运行该条sql，则在.psql_history文件中会有相应语句的记录有密码泄露的风险

注意：

尽量不要使用postgres作为用户密码，防止被攻击。
