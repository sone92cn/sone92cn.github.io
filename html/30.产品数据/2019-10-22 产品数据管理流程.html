产品数据管理流程

一、流程图

mermaid
graph LR
    ST((提出/发现问题)) --分析问题--> R1{是否一次性}
    R1 -- 是 --> R2{可否解决}
    R2 -- 可以 --> S1[尝试解决问题]
    R2 -- 不可以 --> R3{是否可容忍}
    R3 -- 是 --> S2[容忍]
    R3 -- 否 --> R8{上级决策}
    R8 -- 现在处理办法 -->S1
    R8 -- 容忍 --> ED
    R8 -- 无法解决和容忍 --> S15[待以后解决]
    R8 -- 以后解决方案 --> S5
    S2 --> ED
    S1 --成功--> ED((结束))
    S1 --失败--> R3
    R1 -- 否 --> S4[当前+以后]
    S4 -- 当前问题 --> R2
    S4 -- 以后问题 --> R4{可否解决}
    R4 -- 不可以 --> R3
    R4 -- 可以 --> S5[明确解决方案]
    S5 -- 系统解决 --> S6[系统需求]
    S6 --> S7[产品方案]
    S7 --> S8[开发]
    S8 --> S9[测试上线]
    S9 --> R5{是否解决}
    R5 -- 是 --> ED
    R5 -- 否 --> R6[分析原因]
    R6 -- 需求问题 --> S6
    R6 -- 产品方案问题 --> S7
    R6 -- 开发问题 --> S8
    R6 -- 系统无法解决 --> S5
    S5 -- 制度解决 --> S10[制定制度]
    S10 --> S11[发布制度]
    S11 --跟进执行--> R7{是否解决}
    R7 -- 是 --> S12((持续执行))
    R7 -- 否 --> S13[分析原因]
    S13 -- 制度问题 --> S10
    S13 -- 制度无法解决 --> S5
    S5 -- 其它方式解决 --> S14[解决问题]
    S14 --> ED1((结束))
    S5 --尝试后无法解决--> R3

二、流程说明

  1. 产品数据管理的起点是“发现/提出问题”，终点是“问题得到解决”
  2. 系统需求管理的起点是“要解决的问题”，终点是“问题被解决/不能用系统解决”
